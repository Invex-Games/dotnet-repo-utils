Orchestrates the end-to-end breaking change check for a pull request: it locates the most recent release to use as a baseline, identifies breaking changes against it, validates that the version has been bumped appropriately, and reports the result back to GitHub as a check run.

PerformPrBreakingChangeCheck(SemVer, int, IEnumerable<RootedPath>, string, CancellationToken)

Runs the complete breaking change analysis for a pull request and publishes a GitHub check run summarizing the outcome.

Task PerformPrBreakingChangeCheck(SemVer currentVersion, int pullRequestNumber, IEnumerable<RootedPath> filesToCheck, string githubToken, CancellationToken cancellationToken)

The semantic version produced for the current pull request.

The API definition files whose changes should be analyzed.

The GitHub token used to authenticate the check run creation.

cancellationToken CancellationToken

A token used to cancel the operation.

Returns

Task

Remarks

When no previous release can be found, the check is skipped as there is no baseline to compare against. Otherwise a check run is created with a success conclusion when the version has been bumped to cover the detected changes, or failure when a required bump is missing.
